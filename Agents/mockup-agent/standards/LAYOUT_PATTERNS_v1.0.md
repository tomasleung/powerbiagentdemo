# Document Metadata

LAYOUT_PATTERNS

Version

1.0

Owner

Decision-Driven BI Framework

Status

Approved Standard

Purpose

Provide approved storytelling and dashboard layout patterns for the Mockup Agent.

Layout determines:

```text
Where Information Appears

How Information Flows

How Decisions Are Supported
```

Layout does NOT determine:

```text
Colors

Typography

Branding

Styling
```


# SECTION 01 — LAYOUT PHILOSOPHY

## Core Principle

Layout is storytelling.

Users should naturally progress through:

```text
Context

↓

Attention

↓

Decision

↓

Analysis

↓

Trust

↓

Action
```

---

# SECTION 02 — GLOBAL RULES

## Rule 01

Story Drives Layout.

```text
Story

↓

Layout
```

Never:

```text
Layout

↓

Story
```

---

## Rule 02

Preserve Story Sequence.

If the DSC defines:

```text
A

↓

B

↓

C
```

The layout must preserve:

```text
A

↓

B

↓

C
```

---

## Rule 03

Primary Decision Must Appear Early.

Decision support should be visible within the first screen view.

# SECTION 15 — LAYOUT SUCCESS STATEMENT

A layout succeeds when:

```text
Every Section

appears

when the user needs it
```

and:

```text
Users Follow

The Story

Without Thinking About

The Layout
```

The layout becomes:

```text
A Guided Decision Journey
```

rather than:

```text
A Collection Of Report Sections
```
